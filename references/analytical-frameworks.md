# 分析框架参考手册

> 本文档是后续 Agent 构建分析结构时的速查手册。
> 每个框架给出定义、适用场景和输出格式要求。

---

## 1. MECE 原则（相互独立、完全穷尽）

**定义**：将一个问题空间分解为若干子集，满足两个条件：
- **Mutually Exclusive**：子集之间不重叠
- **Collectively Exhaustive**：子集之和覆盖全部问题空间

**适用场景**：Engagement Manager 构建幽灵卡片时的章节拆解。

**检验方法**：
1. 任取两个子集，问"是否存在同时属于两者的元素？"→ 应为否
2. 问"是否存在不属于任何子集的元素？"→ 应为否

**常见 MECE 切分维度**：
- 价值链：上游原材料 → 中游制造 → 下游应用 → 售后服务
- 利益相关者：供应商 / 制造商 / 渠道商 / 终端用户 / 监管方
- 时间轴：短期(0-1年) / 中期(1-3年) / 长期(3-5年+)
- 地理：国内 / 海外（可按区域细分）
- 商业模式：产品收入 / 服务收入 / 平台收入 / 数据收入

## 2. 明托金字塔原理（Pyramid Principle）

**定义**：自上而下的表达结构——先给结论，再给支撑论据，论据之间满足 MECE。

**结构**：
```
        [核心论点]
       /    |    \
   [论据1] [论据2] [论据3]
   /  \    / \    / \
 [事实] [事实] [事实]...
```

**适用场景**：所有 Agent 的输出结构。

**规则**：
- 任何层级的论点必须是对其下级论据的总结
- 同一层级的论据必须逻辑同类（都是原因、都是步骤、或都是分类）
- 行动标题（Action Title）= 动词开头的结论句（如"扩大东南亚市场份额至15%"而非"东南亚市场分析"）

## 3. PESTLE 分析

**定义**：宏观环境六维分析框架。

| 维度 | 英文 | 关键问题 | 典型数据源 |
|------|------|---------|-----------|
| P | Political 政治 | 政策红利/限制？监管趋势？ | 国务院文件、部委政策 |
| E | Economic 经济 | GDP增速？利率？汇率？消费力？ | 统计局、央行数据 |
| S | Social 社会 | 人口结构？消费习惯？文化趋势？ | 人口普查、消费调研 |
| T | Technological 技术 | 技术成熟度？替代技术？ | 专利数据、技术路线图 |
| L | Legal 法律 | 法规约束？合规成本？知识产权？ | 法律法规数据库 |
| E | Environmental 环境 | 碳排放？ESG要求？资源约束？ | ESG报告、环保法规 |

**适用场景**：蓝方和红方的宏观分析层。

**输出格式**：每个维度给出 2-3 个关键发现，每个发现必须附数据源。

## 4. 波特五力分析（Porter's Five Forces）

**定义**：行业竞争结构五维分析。

| 力量 | 关键问题 | 量化指标 |
|------|---------|---------|
| 现有竞争者 | CR3/CR5 集中度？价格战频率？ | 市场份额、HHI指数 |
| 潜在进入者 | 资本壁垒？技术壁垒？牌照壁垒？ | 最低投资额、技术专利数 |
| 替代品威胁 | 替代产品性价比？转换成本？ | 替代品渗透率趋势 |
| 供应商议价力 | 供应集中度？是否有替代供应？ | 关键原材料供应商数量 |
| 买方议价力 | 客户集中度？标准化程度？ | 大客户占比、产品差异化程度 |

**适用场景**：蓝方的行业竞争分析。

**输出格式**：每个力量评定为 强/中/弱，附具体数据支撑。

## 5. SCQ 框架（Situation-Complication-Question）

**定义**：战略报告的开篇结构——用叙事张力引导读者进入核心问题。

| 要素 | 说明 | 示例 |
|------|------|------|
| S - Situation（情境） | 共识性的背景事实 | "中国低空经济政策密集落地，2024年纳入政府工作报告" |
| C - Complication（冲突） | 打破情境的变化/矛盾 | "但产业链商业化路径不清晰，90%的eVTOL企业仍在亏损" |
| Q - Question（疑问） | 由冲突引出的核心决策问题 | "投资者应如何在不确定性中识别真正的结构性机会？" |

**适用场景**：Chief Arbitrator 撰写最终报告的开篇。

**规则**：
- S 必须是读者已知/可验证的事实
- C 必须与 S 形成张力（不是简单的"但是"转折，而是根本性的矛盾）
- Q 必须是可被报告回答的决策问题

## 6. 2×2 优先级矩阵

**定义**：用两个关键维度将议题分为四象限，快速筛选高价值分支。

**行业调研专用矩阵**：

```
            高潜在影响
               │
    ┌──────────┼──────────┐
    │ 战略机会  │ 核心主题  │
    │ (可选展开)│ (必须深研) │
    │          │          │
低──┼──────────┼──────────┼──高
实施 │ 低优先级  │ 快速验证  │ 实施
难度 │ (排除)   │ (附录提及) │ 难度
    │          │          │
    └──────────┼──────────┘
               │
            低潜在影响
```

**适用场景**：Engagement Manager 筛选幽灵卡片中的行动标题。

**输出格式**：每个行动标题标记所在象限：
- `core_theme` → 必须深入研究
- `strategic_opportunity` → 可选展开
- `quick_validation` → 附录简要提及
- `low_priority` → 从大纲中排除

## 7. 事前验尸法（Pre-Mortem Analysis）

**定义**：假设项目/战略已经彻底失败，倒推失败的原因。

**步骤**：
1. 设定前提："假设一年后，该战略遭遇史诗级失败"
2. 列举所有可能的失败原因（不设限，鼓励极端场景）
3. 为每个原因评估发生概率和影响程度
4. 标记当前正向分析中被忽视的风险

**适用场景**：红方 Agent 的核心方法论。

**输出格式**：
```json
{
  "failure_scenario": "描述失败场景",
  "root_causes": [
    {
      "cause": "具体原因",
      "probability": "HIGH/MEDIUM/LOW",
      "impact": "CATASTROPHIC/SEVERE/MODERATE",
      "blind_spot_in_bull_case": true/false,
      "counter_evidence": "支撑该风险的具体数据"
    }
  ]
}
```

## 8. 黑格尔辩证法（Hegelian Dialectic）

**定义**：正题(Thesis) → 反题(Antithesis) → 合题(Synthesis) 的思维运动。

**在本技能中的应用**：
- **正题** = 蓝方的看多逻辑
- **反题** = 红方的做空攻击
- **合题** ≠ 简单折中。合题应该是：
  1. 找到正反双方论点中**正交的真理维度**
  2. 在更高的抽象层次上整合两者
  3. 保留双方有据可依的部分，扬弃缺乏数据支撑的部分

**适用场景**：Chief Arbitrator 的核心工作方法。

**反模式（必须避免）**：
- ❌ "综合来看，机会与风险并存" → 这是废话，不是合题
- ❌ 取蓝方和红方数据的简单平均 → 这是和稀泥
- ✅ "蓝方关于技术成熟度的判断得到数据支撑（引用），但红方指出的政策不确定性同样真实（引用）。两者的正交维度是：技术就绪但商业化路径受政策节奏约束。因此建议采取分阶段进入策略..."

## 9. MAMV 多数决投票（Modified Analytical Majority Voting）

**定义**：处理红蓝双方对同一事实的分歧时的决策机制。

**规则**：
1. 如果双方引用同一数据源但解读不同 → 回到原始数据源验证
2. 如果双方引用不同数据源且数据一致 → 交叉验证通过，采信
3. 如果双方引用不同数据源且数据矛盾 → 标记为"悬置判断(Explicit Suspension)"
4. 如果一方有硬数据、另一方只有推理 → 采信有数据的一方

**适用场景**：Chief Arbitrator 处理分歧时的裁决规则。
